📖 Fundamentos

¿Qué es LangChain?

Framework de código abierto para construir aplicaciones con Modelos de Lenguaje Grande (LLMs). Permite encadenar componentes como prompts, modelos, herramientas y memoria de forma modular.

🧩 Componentes

Componentes Principales

📝

Prompts

Plantillas reutilizables con variables dinámicas. Soportan roles (system, human, AI).

🤖

Models (LLMs)

Interfaz unificada: OpenAI, Anthropic, Google, Ollama. Misma API, cualquier modelo.

🔧

Tools

Funciones externas que el modelo invoca: APIs, cálculos, bases de datos.

🧠

Memory

Mantiene contexto entre conversaciones. Buffers simples o vectoriales.

📊

Output Parsers

Transforman la respuesta al formato deseado: JSON, listas, objetos.

⛓️

Chains (LCEL)

El operador pipe (|) conecta: Prompt → Model → Parser. Composable.

💻 Código

Ejemplo Simple: LangChain + OpenAI

Python — Invocación directa 
# 1. Importar el modelo
from langchain_openai import ChatOpenAI

# 2. Inicializar con configuración
llm = ChatOpenAI(
    model="gpt-4o-mini",   # Modelo rápido y económico
    temperature=0.7        # Creatividad moderada
)

# 3. Invocar directamente
respuesta = llm.invoke("¿Qué es LangChain?")
print(respuesta.content)

💡 .invoke() envía el prompt y espera la respuesta completa.

Python — PromptTemplate 
from langchain_core.prompts import ChatPromptTemplate

# Plantilla reutilizable con variables
prompt = ChatPromptTemplate.from_messages([
    ("system", "Eres experto en {materia}."),
    ("human", "{pregunta}")
])

mensajes = prompt.format_messages(
    materia="inteligencia artificial",
    pregunta="¿Qué es un transformer?"
)

respuesta = llm.invoke(mensajes)

💡 Las {variables} se reemplazan dinámicamente. Reutiliza el prompt con diferentes valores.

Python — Cadena LCEL 
from langchain_core.output_parsers import StrOutputParser

# Componentes individuales
prompt = ChatPromptTemplate.from_template("Explica {concepto} en 3 oraciones.")
llm    = ChatOpenAI(model="gpt-4o-mini")
parser = StrOutputParser()

# Cadena: Prompt → LLM → Parser
cadena = prompt | llm | parser

# Invocar la cadena completa
resultado = cadena.invoke({"concepto": "redes neuronales"})
print(resultado)  # ← ya es un string limpio 🎉

💡 El pipe | conecta componentes. Es el patrón recomendado en 2026.

🔗 LangGraph

¿Qué es LangGraph?

Extensión de LangChain para construir flujos de trabajo con grafos de estados. Ideal para agentes con lógica compleja, decisiones y ciclos.

📊

StateGraph

Estado compartido entre nodos con TypedDict.

🔲

Nodos

Funciones que procesan y actualizan el estado.

➡️

Aristas

Conexiones fijas o condicionales entre nodos.

🔧

ToolNode

Ejecuta herramientas automáticamente.

Python — Agente ReAct (forma rápida) 
from langgraph.prebuilt import create_react_agent
from langchain_core.tools import tool

@tool
def obtener_clima(ciudad: str) -> str:
    """Obtiene el clima de una ciudad."""
    return f"☀️ Soleado, 25°C en {ciudad}"

agente = create_react_agent(modelo, [obtener_clima])
resultado = agente.invoke({"messages": [("user", "¿Clima en Madrid?")]})

Grafo Manual con StateGraph

Python — StateGraph personalizado 
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
from langgraph.prebuilt import ToolNode

# 1. Definir estado con TypedDict
class Estado(TypedDict):
    messages: Annotated[list, add_messages]  # ← se agregan, no reemplazan

# 2. Nodo agente: llama al modelo con herramientas
def nodo_agente(estado):
    respuesta = modelo_con_tools.invoke(estado["messages"])
    return {"messages": [respuesta]}

# 3. Enrutamiento: ¿tiene tool_calls → tools, si no → END
def decidir(estado) -> Literal["tools", "__end__"]:
    ultimo = estado["messages"][-1]
    if hasattr(ultimo, "tool_calls") and ultimo.tool_calls:
        return "tools"
    return "__end__"

# 4. Construir y compilar
grafo = StateGraph(Estado)
grafo.add_node("agente", nodo_agente)
grafo.add_node("tools", ToolNode(herramientas))
grafo.add_edge(START, "agente")
grafo.add_conditional_edges("agente", decidir)
grafo.add_edge("tools", "agente")  # ← ciclo: tools → agente
app = grafo.compile()

Bucle Agente ↔ Tools
El agente llama herramientas hasta terminar

💻 Código Multi-Agente

Código del Supervisor

Python — Sistema Multi-Agente 
# Estado con campo de enrutamiento
class EstadoMultiAgente(TypedDict):
    messages: Annotated[list[BaseMessage], add_messages]
    siguiente_agente: str  # ← el supervisor escribe aquí

# Supervisor: analiza y decide
def nodo_supervisor(estado):
    prompt = "Elige: calculadora, investigador, analista o FINALIZAR"
    decision = modelo.invoke(prompt)
    return {"siguiente_agente": decision}

# Enrutamiento condicional
def enrutar(estado) -> str:
    rutas = {"calculadora": "calc", "investigador": "inv", "analista": "an"}
    return rutas.get(estado["siguiente_agente"], "final")

# Grafo completo
grafo = StateGraph(EstadoMultiAgente)
grafo.add_node("supervisor", nodo_supervisor)
grafo.add_node("calc", nodo_calculadora)
grafo.add_node("inv", nodo_investigador)
grafo.add_node("an", nodo_analista)
grafo.add_node("final", nodo_respuesta_final)

grafo.add_edge(START, "supervisor")
grafo.add_conditional_edges("supervisor", enrutar)
grafo.add_edge("calc", "supervisor")  # ← ciclos de retorno
grafo.add_edge("inv", "supervisor")
grafo.add_edge("an", "supervisor")
grafo.add_edge("final", END)
app = grafo.compile()
🧠 Memoria

Memoria Conversacional

LangGraph persiste el estado entre invocaciones usando Checkpointers. Cada conversación se aísla con un thread_id único.

Python — Memoria con MemorySaver 
from langgraph.checkpoint.memory import MemorySaver

# 1. Crear el checkpointer
memoria = MemorySaver()

# 2. Compilar con checkpointer
app = grafo.compile(checkpointer=memoria)

# 3. Cada invocación usa un thread_id
config = {"configurable": {"thread_id": "user-123"}}

# Mensaje 1: se presenta
app.invoke({"messages": ["Soy Sebastián"]}, config)

# Mensaje 2: ¡RECUERDA el nombre!
app.invoke({"messages": ["¿Cómo me llamo?"]}, config)
# → "Te llamas Sebastián"
💾

MemorySaver

Guarda estado en RAM. Ideal para desarrollo. En producción: PostgresSaver, SqliteSaver.

🔑

thread_id

Identificador único de conversación. Mismo ID = misma memoria. Diferente ID = memoria aislada.

🔄

Persistencia Automática

El checkpointer guarda el estado después de cada .invoke(). No necesitas gestionar la memoria manualmente.

🎯 ¿Cuál elegir?

LangChain si necesitas:

  • RAG / búsqueda semántica
  • Cadenas de prompts
  • Múltiples integraciones

LangGraph si necesitas:

  • Agentes con lógica compleja
  • Flujos con ciclos
  • Sistemas multi-agente

OpenAI SDK si necesitas:

  • Llamadas directas simples
  • Mínima abstracción
  • Máximo rendimiento
